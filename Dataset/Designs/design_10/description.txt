MODULE SPECIFICATION: if_stage (Instruction Fetch Stage)

1. ARCHITECTURAL INTENT

    Type: Instruction Fetch (IF) Unit for a processor pipeline.

    Active Configuration: This design implements a simple sequential instruction fetch mechanism with a single Program Counter (PC) register and no control flow logic (no branching, jumping, or stalling). The PC advances strictly by 4 bytes every clock cycle.

    Mechanism: The module maintains a Program Counter that feeds into instruction memory to fetch a 32-bit instruction. A combinational adder computes the next PC (PC + 4), which is fed back into the PC register on the next clock edge.

    Execution Model: One instruction is fetched per clock cycle. The instruction memory is word-aligned and accessed combinationally using the current PC value.


2. PORT INTERFACE

    Global:
        clk   : System clock (rising-edge triggered).
        reset : Active-high synchronous reset.

    Outputs:
        pc    [31:0] : Current Program Counter value (instruction address).
        instr [31:0] : Instruction fetched from memory at address pc.


3. CRITICAL INTERNAL STATE (Registers & Wires)

    pc (Register):
        A 32-bit register that holds the current instruction address.
        On reset, it is initialized to 0.
        Otherwise, it updates to pc_next on every rising clock edge.

    pc_next (Wire):
        The next value of the Program Counter.
        Currently always assigned as pc + 4.

    pc_plus4 (Wire):
        Output of the adder that computes the next sequential instruction address.

    mem (Memory Array):
        A 256 x 32-bit instruction memory array.
        Initialized with NOP instructions and optionally overwritten using a HEX file.


4. INTERNAL MODULES

    pc (Sub-module):
        A synchronous register block that updates the PC value.
        Behavior:
            if (reset) pc <= 0;
            else       pc <= pc_next;

    pc_adder (Sub-module):
        A combinational adder that increments the PC by 4.
        Implements:
            pc_plus4 = pc + 32'd4;

    instr_mem (Sub-module):
        A word-addressable instruction memory.
        - Depth: 256 words
        - Width: 32 bits
        - Initialization:
            * All locations set to NOP (32'h00000013)
            * Program loaded via $readmemh
        - Access:
            instr = mem[addr[9:2]]
        Lower two bits of the address are ignored to enforce word alignment.


5. BEHAVIORAL CONSTRAINTS & DEBUGGING CONTEXT

    Sequential Fetch Only:
        The design always executes instructions sequentially.
        No support for branch, jump, or pipeline control logic exists.

    Address Alignment:
        Instruction memory uses addr[9:2], meaning:
            - Addresses must be word-aligned
            - Misaligned accesses will be truncated, potentially causing incorrect instruction fetch

    Reset Behavior:
        On reset, PC is set to 0.
        Execution begins from mem[0].

    Memory Initialization Dependency:
        Correct functionality depends on successful loading of "program.hex".
        If loading fails, the processor will execute NOP instructions.

    Timing Behavior:
        - PC updates on the rising edge of the clock.
        - Instruction memory is combinational.
        - instr reflects the instruction at the current PC within the same cycle.
        - No pipeline latency is present in this stage.

    Memory Size Limitation:
        - Only 256 instructions can be stored.
        - Address range effectively limited to 1024 bytes.


6. DESIGN LIMITATIONS & EXTENSIBILITY

    Control Flow Limitation:
        pc_next is hardwired to pc_plus4.
        To support branching or jumping, a multiplexer must be introduced.

    Pipeline Integration:
        This module does not include pipeline registers (e.g., IF/ID).
        External logic is required for pipelined CPU integration.

    Scalability:
        Memory depth and addressing are fixed.
        Expansion requires modifying address indexing and memory size.

    Future Enhancements:
        - Add branch/jump logic using a PC selection multiplexer
        - Introduce stall and flush control for pipeline support
        - Replace instruction memory with cache for performance optimization
